How these components work

  • Composition: built from standard Horizon now-* components, customized via properties / slots / design tokens — never hand-rolled.
  • Configuration: every property below is set in UI Builder and bindable to data resources / client state.
  • Events: declared as actions so they appear in the UI Builder Events tab. Only events an external page needs to bind are exposed — purely internal UI state isn't.

Standard components used: none — fully custom
